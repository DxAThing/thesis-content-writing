# Storyline Templates for Common Paper Types

## Type 1: "We solve an existing problem better"

```
Task is important → current SOTA has limitation X → root cause is Y →
we propose method Z that addresses Y → advantage 1, advantage 2 →
experiments show improvement on standard benchmarks
```

Key challenge: Making the contribution feel non-trivial, not just "we tuned hyperparameters."

## Type 2: "We define a new problem"

```
General area is important → we identify a specific, unstudied setting →
this setting is challenging because A, B, C →
we propose a pipeline with modules for each challenge →
we create a benchmark and demonstrate strong baseline results
```

Key challenge: Convincing reviewers the new problem is worth studying.

## Type 3: "We bring an insight from domain A to domain B"

```
Domain B has challenge X → in domain A, a similar challenge was solved by insight I →
direct transfer does not work because of difference D →
we adapt insight I with modifications M1, M2 →
experiments show this cross-domain transfer is effective
```

Key challenge: Showing the adaptation is non-trivial.

## Type 4: "We unify previously separate approaches"

```
Approach A has strength S1 but weakness W1 →
Approach B has strength S2 but weakness W2 →
we propose a unified framework that combines S1 and S2 →
the key challenge in unification is U →
we solve U via contribution C →
experiments show unified approach > both individuals
```

Key challenge: The unification must produce synergy, not just aggregation.

## Type 5: "We discover something surprising"

```
Common belief B says X about task T →
we observe through experiment/analysis that actually Y →
this observation leads to design principle P →
we propose method M based on P →
experiments confirm the observation and the method effectiveness
```

Key challenge: The observation must be genuinely surprising and verifiable.
