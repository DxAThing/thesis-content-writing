# research-and-writing · AI 学术写作技能

一个覆盖**科研全流程**的 Copilot 技能，从最初的选题灵感到最终的投稿润色，每个阶段都有对应的引导和模板支持。同时支持**中文学位论文**和**英文会议/期刊论文**两条写作路线。

---

## 能做什么

### 一、研究选题与创意孵化

帮你系统整理某个研究方向的文献格局，定位尚未被解决的问题（GAP），并生成候选研究想法。

- **文献格局梳理**：按技术路线分类整理核心论文，标注每篇的贡献、局限和方法特点
- **GAP 分析**：识别现有方法共同未解决的挑战，给出量化评分
- **Idea 评估**：从学术价值、技术可行性、资源适配三个维度给研究想法打分
- **Idea Card 输出**：最终生成结构化的研究想法卡片，包含问题定义、解决思路、潜在实验方案

---

### 二、研究规划（Research PRD）

把模糊的研究方向转化为可执行的技术方案。

- **三维诊断**：学术价值 × 技术可行性 × 资源约束，综合评估方向是否值得投入
- **v1 概念文档**：明确研究方向和核心贡献（需用户确认后再进入下一阶段）
- **v2 落地文档**：可执行的技术路线、实验设计和时间规划

---

### 三、论文故事线设计

在动笔之前，先把论文的逻辑骨架搭好。

> **中文学位论文章节安排前置建议**
>
> 在 AI 帮你规划章节结构之前，强烈建议先向导师或同课题组的师兄师姐索要他们已完成的毕设/学位论文。这些论文反映了你所在院校和导师对章节安排的实际期望，是最可靠的参考依据。AI 给出的章节骨架应与参考论文对照后再确认，不建议直接跳过这一步。

- **逆向推理**：从"我们的贡献是什么"出发，倒推论文需要建立什么论证链
- **Introduction 逻辑图**：提供 13 种 Introduction 模板组合（任务引入 + 挑战描述 + 方案呈现），覆盖常见论文类型
- **全文大纲**：每个章节的核心论点 + 段落分工
- **Claim-Evidence 预映射**：提前标出每个核心声明对应的实验证据来源

---

### 四、实验规划

- **实验设计矩阵**：将论文中每个声明（Claim）映射到对应的验证实验
- **消融实验清单**：从模块依赖关系自动推导消融方案，避免遗漏
- **实验记录模板**：结构化的实验日志，贯穿整个研究过程持续更新

---

### 五、逐章节写作

支持从零起草或在已有草稿上继续写作，所有章节均有专属引导：

| 章节 | 支持内容 |
|------|----------|
| **摘要** | 3 种模板（挑战驱动 / 贡献驱动 / Insight 驱动） |
| **Introduction** | 逆向推理框架 + 13 种模板示例 |
| **Related Work** | 多种组织策略，避免"流水账"式罗列 |
| **Method** | 模块三要素结构（动机 → 设计 → 技术优势） |
| **Experiments** | 3 个核心问题框架 + 表格规范 |
| **Conclusion** | 避免空洞收尾的写法建议 |

> **语言路线**：写作前会先询问你是中文学位论文还是英文投稿论文，分别套用对应的规范。


---

### 六、润色与去 AI 味

- **保守润色（Refine）**：修正语法、逻辑、学术语气，保留作者原意，不乱改
- **去 AI 味（Humanize / De-AI）**：去除空洞套话、机械对仗、模板感段落，同时保留专业术语
- **逻辑检查**：识别矛盾表述、未定义符号、声明-证据不匹配、跨节命名不一致等实质性问题

---

### 七、翻译与长度调整

- **中→英**：学术改写为 LaTeX 格式，遵循英文投稿规范
- **英→中**：忠实直译，用于理解和参考
- **中→中**：口语草稿 → 正式学术中文
- **缩写**：在不损失信息的前提下压缩 5–15 词
- **扩写**：补全隐含逻辑，扩展 5–15 词

---

### 八、图表辅助

- **架构图设计**：给出 Mermaid / DrawIO / SVG 的绘图简报，包含插入位置、视觉要素、图题建议
- **实验可视化**：根据数据类型推荐合适的图表类型（折线图、热力图、柱状图等）
- **图/表标题**：符合期刊规范的图标题和表标题写法

---

### 九、Claim-Evidence 对齐核查

确保论文中每一个声明都有实验数据支撑。

- **Build 模式**：从摘要和 Introduction 提取所有声明，建立映射表
- **Verify 模式**：逐条检查证据充分性和一致性
- **Fix 模式**：对无支撑声明进行弱化措辞，或标记需要补做的实验

---

### 十、审稿与审核-精修循环

- **5 维度审稿（英文）**：贡献新颖性 / 写作清晰度 / 实验强度 / 评估完整性 / 方法合理性
- **审稿人模拟**：以目标会议/期刊审稿人视角给出 Summary、Strengths、Weaknesses 和评分建议
- **6-Pass 审核-精修循环**：阅读流 → 去 AI 味 → 规范检查 → 引用核实 → 逻辑检查 → 审稿人视角，逐轮直到通过

## 使用方式

在 VS Code Copilot Chat 中，直接用自然语言描述你的需求即可触发此技能，例如：

```
帮我分析这个方向的文献格局，找找有没有可以做的 gap
帮我设计论文的逻辑骨架，我的工作是 XXX
帮我起草 Introduction，目标是 ACL 2025
润色这段方法描述，去掉 AI 味
帮我检查 claim 和实验的对应关系
以审稿人视角给我的论文提意见
```

## 适用人群

- 准备撰写学位论文（本、硕、博）的学生
- 向 NLP / CV / AI / 系统 等方向的会议/期刊投稿的研究者
- 需要从头规划课题、设计实验的早期研究者
- 已有草稿、需要系统润色和自查的作者
